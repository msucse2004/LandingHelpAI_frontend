Survey Builder

고객 “진단 흐름”(survey)과 추천 규칙(recommendation)을 함께 설계·관리합니다.

1) Questionnaire 목록

survey = 고객이 답하는 진단 프로세스입니다. 질문/옵션 구조를 먼저 만들고, 오른쪽에서 추천 규칙(패키지/모듈/애드온 연결)을 분리해서 정의하세요.

Name Code Active version Active Actions

질문지를 선택하면 아래에서 빌더를 편집할 수 있습니다.

2) Survey 구조 빌더

기본 정보/버전/질문/옵션을 관리합니다. (질문=고객이 답하는 단계, 옵션=선택지)

중복 없이 식별용 코드입니다.

-

버전 탭에서 “Publish” 및 “Mark active”를 수행하세요.

3) Version 관리

DRAFT(작성 중) → PUBLISHED(고정) → Active 버전으로 설정합니다.

# Status Published Active Actions
버전 액션

버전/질문지를 선택해야 버튼이 활성화됩니다.

4) Question builder

각 질문은 고객 진단 흐름의 단계입니다.

Order Section Code Label Required Active Actions
질문 편집

전화/대면이 아닌 “설문 답변” 기반으로 추천이 달라집니다.

간단 예: {"type":"question_option_equals","question_code":"company_size","option_value":"small"}

5) Option builder

선택지는 질문(item)에 종속됩니다. 옵션 삭제는 안전성 검사를 거칠 수 있습니다.

Order Value Label Actions
옵션 편집

6) Recommendation rules

규칙은 survey 답변(questions/options)으로 조건을 만들고, 그 결과로 추천할 package/module/add-on을 연결합니다.

Priority Active Result Condition Actions

백엔드 스키마의 result_code는 코드 문자열입니다.
