회원 초대 메일

받는 사람에게 가입 링크가 담긴 메일을 보냅니다. 상대는 링크에서 회원가입을 마친 뒤 이메일 인증을 완료해야 정상적으로 로그인할 수 있습니다. 역할(권한)은 여기서 선택한 값으로 고정됩니다.

이용 흐름 (받는 사람 관점)

  1. 메일함에서 초대 메일을 연다. 스팸·프로모션함도 함께 확인하도록 안내하세요.
  2. 메일 속 «가입하기» 링크를 눌러 회원가입 페이지로 이동한다.
  3. 이메일과 역할은 초대와 맞춰져 있으므로, 아이디·이름·비밀번호 등만 입력한다.
  4. 가입 후 도착하는 인증 메일의 링크로 이메일을 확인한다.

이미 같은 이메일로 가입된 사람에게는 초대를 보낼 수 없습니다. 그 경우 기존 계정으로 로그인·비밀번호 찾기를 안내하세요.

보안: 현재 관리 API는 데모용으로 보호되지 않습니다. 실서비스에서는 관리자 인증·감사 로그를 붙이세요.

운영자 체크리스트

  • 받는 사람 이메일 주소를 다시 한 번 확인합니다 (오타 시 다른 사람에게 권한이 갈 수 있습니다).
  • 역할은 실제 업무에 맞는 최소 권한부터 선택합니다.
  • 선택 메시지에는 회사명·담당자·다음 할 일 등 짧게 적으면 수신자가 맥락을 이해하기 쉽습니다.
  • 외부 사람에게 보낼 때는 전송 결과의 가입 링크가 https://… 인지 확인합니다. 계속 127.0.0.1이면 관리자 로그인 상태에서 API GET /api/admin/runtime/public-app-base 로 어떤 후보가 선택됐는지 확인하고, LandingHelpAI_backend/.env 의 예전 localhost 줄을 지우거나 Docker Compose environment로 공개 URL을 넣은 뒤 백엔드를 재시작하세요.

초대 보내기

역할 코드 옆 한글 이름을 참고해 선택하세요. 파트너는 로그인 권한(RBAC)이며, 아래 «파트너 역할/유형»은 업무 매칭용 카탈로그 값입니다.

파트너 역할/유형은 초대에만 저장되며, 가입 완료 후 파트너 프로필에 반영되어 워크플로우 매칭에 사용됩니다. 이름·전화·지역은 초대받은 분이 회원가입 화면에서 입력합니다.

워크플로 단계와 별개로, 파트너 계정의 동작 모드를 구분합니다.

기본값은 BOTH입니다.
